# Pólya - "How to Solve It" (1945)

## Core Framework: 4 Phases

1. **Understand the Problem** - What is the unknown? What are the data? What is the condition?
2. **Devise a Plan** - Find the connection between data and unknown. Look for related problems, analogous problems, decomposable subproblems.
3. **Carry Out the Plan** - Check each step. Can you prove it is correct?
4. **Look Back** - Can you check the result? Can you derive it differently? Can you use the result or method for another problem?

## Key Heuristics (Dictionary)

| Heuristic | Description |
|-----------|-------------|
| **Analogy** | Find a problem analogous to yours and solve that |
| **Auxiliary Elements** | Add new elements to get closer to solution |
| **Generalization** | Find a more general problem |
| **Specialization** | Find a more specialized problem |
| **Induction** | Solve by deriving generalization from examples |
| **Variation of the Problem** | Change the problem to create new ones whose solutions help |
| **Auxiliary Problem** | Find a subproblem whose solution helps |
| **Decomposing & Recombining** | Decompose and recombine elements in new ways |
| **Working Backward** | Start from goal and work backward |
| **Draw a Figure** | Visual representation |

## Key Quotes

- "It is foolish to answer a question that you do not understand."
- "When the approach is right, the student smells the solution, he feels it."
- "Problems proposed and solutions found are the mathematical substance."

## Implications for IntuitMath

- The 4-phase model maps naturally to our workflow: Motivation -> Derivation -> Verification -> Reflection
- Pólya's heuristics should be embedded as searchable patterns in the cross-domain index
- The "Look Back" phase is underemphasized in most math education - our skill should make it prominent
- The emphasis on analogy and auxiliary problems aligns with the multi-perspective paradigm
