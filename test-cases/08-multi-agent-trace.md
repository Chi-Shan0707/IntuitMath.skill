# Test Case: Useful-Not-Forced Multi-Agent Trace

**Question**: "Give a deep explanation of compactness. Why is it the right replacement for finiteness?"

## Expected Pattern

Because this is deep, historical, conceptual, and cross-domain, multi-agent mode
is useful if available. The answer should not use agents by ritual; it should
show why the roles add value.

## Required Role Outputs

- **Historian**: origin pressure from analysis/topology and labels for verified
  history vs pedagogical reconstruction.
- **Intuition Builder**: finite-subcover toy example and sequence examples.
- **Skeptic**: counterexamples such as closed-but-not-bounded or bounded-but-not
  closed subsets of \(\mathbb{R}\), and non-Euclidean/topological caveats.
- **Formalizer**: open-cover definition plus metric-space sequential
  compactness equivalence with assumptions.
- **Connector**: at least one clarifying connection, such as optimization
  existence theorems, Arzelà-Ascoli, logic compactness, or probability tightness.
- **Synthesizer**: final answer in discovery order.

## Key Elements to Check

- [ ] Explains why multi-agent is useful for this task.
- [ ] Does not spawn/use roles for trivial subparts.
- [ ] Contains counterexamples and repaired definitions.
- [ ] Separates Euclidean Heine-Borel from general topological compactness.
- [ ] Cross-domain connection clarifies mechanism, not just application.
